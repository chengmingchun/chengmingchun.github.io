Rust 入门：所有权系统详解

Rust 最令人着迷的特性之一就是它的所有权系统（Ownership System）。它让 Rust 无需垃圾回收器就能保证内存安全，这在系统编程语言中是极为罕见的。

核心三原则

  1. Rust 中每个值都有一个所有者（owner）
  2. 同一时刻只能有一个所有者
  3. 当所有者离开作用域，值就会被自动释放

移动语义（Move Semantics）

let s1 = String::from("hello");
let s2 = s1; // s1 的所有权移动到 s2

// println!("{}", s1); // ❌ 编译错误！s1 已失效
println!("{}", s2); // ✅ 正常

这与 C++ 的浅拷贝不同——Rust 在编译期就阻止了悬空指针的产生。

借用（Borrowing）

当你不想转移所有权时，可以使用借用：

fn calculate_length(s: &String) -> usize {
    s.len() // 只是借用，不拥有
}

let s1 = String::from("hello");
let len = calculate_length(&s1);
println!("'{}' 的长度是 {}", s1, len); // s1 依然有效

可变借用的规则

let mut s = String::from("hello");

let r1 = &mut s;
// let r2 = &mut s; // ❌ 同一时刻只能有一个可变借用！

r1.push_str(", world");
println!("{}", r1);

这个规则从根本上消除了数据竞争的可能性。

生命周期（Lifetimes）

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

生命周期注解告诉编译器引用之间的关系，确保返回的引用始终有效。

总结

Rust 的所有权系统是一套静态分析规则，在编译期消除了：

  • 悬空指针（Dangling pointers）
  • 双重释放（Double free）
  • 数据竞争（Data races）
  • 使用已释放内存（Use-after-free）

初学时会感觉“编译器在和你作对“，但掌握之后你会发现这是世界上最好的代码审查员。
