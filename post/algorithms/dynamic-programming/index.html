动态规划：从记忆化搜索到状态转移

动态规划（Dynamic Programming，DP）是算法面试中的高频考点，也是很多人的“噩梦“。本文尝试用直觉性的方式解释它的本质。

DP 的本质

DP = 穷举 + 去重

所有 DP 问题本质上都是穷举，但通过记录已计算的子问题结果来避免重复计算，从而将指数级复杂度降到多项式级。

经典问题：斐波那契数列

暴力递归（指数级）

def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
# 时间复杂度：O(2^n) — 灾难性的

记忆化搜索（自顶向下）

def fib(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]
# 时间复杂度：O(n) ✅

状态转移（自底向上）

def fib(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
# 时间复杂度：O(n)，空间复杂度：O(n)

空间优化版

def fib(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr
# 空间复杂度：O(1) 🚀

三步解题框架

  1. 定义状态：dp[i] 代表什么？
  2. 写出转移方程：dp[i] 怎么从之前的状态推导？
  3. 确定初始值和遍历顺序

经典例题：0/1 背包

def knapsack(weights, values, capacity):
    n = len(weights)
    # dp[i][j] = 前i个物品，容量为j时的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for j in range(capacity + 1):
            # 不放第i个物品
            dp[i][j] = dp[i-1][j]
            # 放第i个物品（如果放得下）
            if j >= weights[i-1]:
                dp[i][j] = max(dp[i][j],
                               dp[i-1][j - weights[i-1]] + values[i-1])
    
    return dp[n][capacity]

小结

类型核心思路代表题目
线性DP当前状态依赖前几个状态爬楼梯、打家劫舍
区间DP从小区间推大区间戳气球、最长回文子序列
背包DP选或不选、装或不装零钱兑换、分割等和子集
状态机DP状态之间相互转换买卖股票系列

掌握 DP 的关键在于大量练习，让自己对“状态“的感觉越来越敏锐。
