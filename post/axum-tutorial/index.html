极简入坑指南：用 Rust Axum 优雅托管 API 与前端博客

提到 Rust 后端，很多人会被复杂的生命周期和晦涩的数据结构劝退。但实际上，借助当前最强大的生态框架之一 Axum，搭建一个同时能提供数据接口（API）和托管前端静态网站的服务，简直像喝水一样自然。

今天，我们就以本博客的后端实现为例，带你拆解这个优雅、极速的“双轨制”后端代码。

1. 为什么是 Axum？

Axum 是由官方 Tokio 异步框架团队开发的原生 Web 框架。它的核心理念就是利用 Router（路由）组装各类服务。更棒的是，它能够和塔层（tower_http）完美结合，让你轻松拥有文件托管、CORS 跨域、日志记录等企业级功能。

2. 核心架构拆解

我们的目标很简单：

  1. 提供接口：通过 /api/posts 返回全部文章信息。
  2. 托管前端：把普通的 HTML / CSS / JS 原样发给客户端（浏览器）。

第一步：设置路由树 (Router)

在 Axum 中，构建后端其实就是在拼装一棵“路由树”。

use axum::{routing::get, Router};

// 第一步：定义你的路由
let app = Router::new()
    // 凡是访问 /api/posts 的，都交给 api_posts 这个函数处理
    .route("/api/posts", get(api_posts))
    // 文章内容的动态抓取接口
    .route("/api/posts/*slug", get(api_post));

这里的 api_posts 是一个普通的异步函数，它会扫描本地的 posts 目录解析 markdown，并将其自动打包为 JSON 发送给前端：

use axum::Json;

// Axum 会自动把返回的结构体序列化成标准的 JSON 格式！
async fn api_posts() -> Json<serde_json::Value> {
    let posts = scan_my_markdown_files(); 
    Json(serde_json::json!({ "posts": posts }))
}

第二步：用 ServeDir 优雅托管前端

如果只有后端接口，显然不够。我们要把编译好的前端页面（如本博客的 index.html，style.css）展示给用户。

对于小白来说，不用自己手写读取文件内容的函数！tower_http 中的 ServeDir 就是为此而生的：

use tower_http::services::ServeDir;

let app = Router::new()
    .route("/api/posts", get(api_posts))
    // 神来之笔：把没被上面拦截的所有路径，都指向本地的 "./static" 文件夹
    .nest_service("/", ServeDir::new("static").fallback(ServeDir::new("static")));

这段代码极为优雅：

  • 只要请求没有匹配到 /api/ 路径，Axum 就会自动去找你本地当前目录下的 static/ 文件夹。
  • 比如浏览器访问 /style.css，就会自动返回 static/style.css，连 MIME 类型都会自动帮你算好！

第三步：起爆服务器

最后，我们需要绑定并监听一个网络端口（例如 0.0.0.0:3000），把刚才的 app 交给服务器去跑：

#[tokio::main] // 启动 Tokio 异步运行时
async fn main() {
    // 省略刚才我们组装 app 的代码...

    // 绑定本地端口 3000
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000")
        .await
        .expect("无法绑定端口");

    println!("🚀 Blog 服务器运行在 http://localhost:3000");
    
    // 开始接收用户的网络请求并处理
    axum::serve(listener, app).await.unwrap();
}

3. 把状态安全地传递给后端 (State)

有时候我们需要保持某种状态（比如：博客文章目录的具体路径，或者数据库连接）。在 Node/Express 中你可能直接定义一个全局变量，但这在 Rust 的多线程环境下是不安全的。

Axum 提供极好的 State 支持：

// 定义你的状态
struct AppState {
    posts_dir: PathBuf,
}

// 把它包裹在线程安全的 Arc 共享指针里
let state = Arc::new(AppState { posts_dir: PathBuf::from("posts") });

// 把它注入给 Router
let app = Router::new()
    ...
    .with_state(state);

当你在 api_posts 里需要这个路径时，只需要在函数签名里加一个提取器 (Extractor)：

use axum::extract::State;

async fn api_posts(State(state): State<Arc<AppState>>) -> Json<serde_json::Value> {
    // 这里你直接可以通过 state.posts_dir 安全地获取目录啦
}

总结

这就是全部魔法了！ 没有繁琐的配置文件，没有庞大厚重的类对象体系，我们只需要 Router、Handler 聚合数据、ServeDir 兜底前端。几十行极简代码，我们就能完美承托起这个运行如飞的全栈小应用。
